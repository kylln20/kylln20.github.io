ArrayLists

Basics

Arraylists, as the name suggests, are similar to arrays. There are some key differences though.
Unlike arrays, arraylists have a DYNAMIC memory space. This means: one can change the number of values stored in the array, removing and adding “spaces”, or indexes instead of just changing existing values. While this does lead arraylists to take longer time to process, they can also be more convenient.

  • The number of "spaces" in an array is called the capacity of the arraylist
  • The number of filled spaces in an arraylist is called the size of the arraylist
The elements stored in an arraylist must be OBJECTS, or not primitive data types. So arraylists cannot be declared to store int, float, or boolean type values, among other data types. Thankfully, for each of these data types, Java has object classes that can be automatically converted into these primitive data types.
  • int can be converted to and converted from the Integer class
  • double can be converted to and converted from the Double class
  • boolean can be converted to and from the Boolean class

Syntax

ArrayLists are generally initialized as ArrayList name = new ArrayList(opt. size). Within the brackets, one can specify the starting capacity. If it isn’t specified, then the default capacity is set to 10. When initialized this way, none of the “spaces” within the capacity actually have values. These values are assigned with ArrayList methods, to be mentioned later.

/*
 To make an arraylist to store whole number values:
 since ints are a primitive data type, the Integer class is used */
ArrayList<Integer> numbers = new ArrayList<Integer>();

/*
 To make an arraylist to store words:
 since 5 was added as a parameter, the capacity of the arraylist is set to 5 "spaces."
 By default, this capacity is set to 10
 If by adding elements, the size exceeds the capacity, the capacity will by default double */
ArrayList<String> words = new ArrayList<String>(5);

/*
 To make an arraylist to store Cats (a self-written class in the OOP tutorial example):
*/
ArrayList<Cat> cats = new ArrayList<Cat>();

Arrays Vs ArrayLists

Similarities

Both arraylists and arrays are used to:
  • store multiple values, known as elements, under one variable name
  • access and change values with indexes/indices that start at 0
  • store loop traversable data for convenience (one has to be careful with arraylists, more on that later

Differences

Arrays ArrayLists
Have a fixed length, accessed by .length Have a dynamic length, accessed by .size()
Can store primitive and object data types Can only store object data types
Generally traversed with a for loop or a for each loop Easier to traversed with a while loop and a iterator/index-counter

ArrayList Methods

Name Description
size() returns the size of the arraylist (int)
isEmpty() returns whether or not the arraylist is empty (boolean)
add( [opt int i], Object obj) inserts the Object obj at the i-th index, and pushes all other elements to the right. If i is not given, then it adds obj to the end of the arraylist
set(int i, Object obj) sets the element at index i to the value obj, and returns the old value at index i
get(int i) returns the element at index i
remove(int i) removes the element at index i, as well as returns it
contains(Object obj) returns whether or not obj is located in the arraylist (boolean)
indexOf(Object obj) returns the index of the first occurence of the Object obj in the arraylist (int). If obj is not present in the arraylist, then it returns -1.

Traversal

Since one can remove and add elements to an arraylist, that means it's length as well as the indexes of values can change. This makes using for or for-each loops difficult, since both can't (conveniently) keep track of shifting elements.
For and for-each loops can still be used, but (for-each loops especially) should only be used in instances where no elements are being removed.
