Basics of OOP

Coding and Reality

Object Oriented Programming: A method of programming based on the concept of “Objects” which are meant to reflect objects, (or things, or nouns) that exist in real life

Objects in real life have

  • a name
  • characteristics
  • behaviours
Objects in code have

  • an identity
  • variables
  • methods
This represents

  • a space in memory
  • properties/data
  • interactions with the object

Parts of a Class

/**A class that represents a cat*/

public class Cat{ /** identity */
/** characteristics */
 private String name;
 private int age;
 private double weight; //in pounds
 private String furColour;
 private boolean vaccinated;

/** behaviours*/
 public void vaccinate(){
  vaccinated = true;
 }
 public void birthday(){
  age++;
 }

/** in the cat class, there needs to be a constructor*/
 public Cat(String n){
  name = n;
  age = 1;
  weight = 0.22;
  furColour = "orange";
  vaccinated = false;
 }

 public Cat(){
  name = "kitty";
  age = 1;
  weight = 0.22;
  furColour = "orange";
  vaccinated = false;
 }
}


/** In a seperate class, an object of the cat class can be made*/
Cat kitty = new Cat("MiMi");
Cat ashy; /** This object does not call the constructor, and so it currently has NULL value*/ 
A Class is a template of an object

  • An object is an instance of a class
  • It has template variables and methods that can be used
  • These may be accessible without making an instance (labelled with static)
  • Or they may only be accessible only by making an object of that class (known as instance variables/methods)
To assign values:

Constructor - a special kind of “method” => same name as the class, no return type specified
  • a constructor creates an objects
  • it can have parameters
  • there can be multiple constructors, with different inputted parameters
  • similar to overloaded methods
When you create a object, you must either first
  • initialize it
  • declare an instance of that class
    • default class value is NULL

Using an Object/Class

Dot Notation - objectName.method() or objectName.variable
  • methods and variables are called members of an object, because they are the data that is contained in the class
    • members are called on using dot notation
    • they might be called on to change their value (variables) or to borrow their function (methods)
  • some objects however, can’t have their data changed after construction

Representing a Class

UML (Unified Modelling Language) is a standard format used to represent classes
  • ( - ) indicates private members, usually variables
  • ( + ) indicates public members, usually methods
  • variables are represented as
    • +/- varName: dataType
  • methods are represented as
    • +/- methodName(paramName: dataType): returnType

Inheritance

Definition

  • Inheritance is when a class takes on the members (methods and variables) of another class, referred to as its parent class (aka a superclass)
  • Java classes can only inherit from one class
    • but can have a very long chain of extended classes
  • All java classes by default inherit from the Object class (a thing that exists in java)

Implementing Inheritence

public class Subclass extends Superclass{
 public Subclass(){
  super();
 }
}

super(); calls on the constructor of the extended class
  • if used, must be the first line in the subclass constructor
  • the superclass (Superclass) must at least have a no-param constructor to avoid errors

The "this" keyword

this keyword - Used to refer to elements of the class being worked in, especially when other elements have the same name

It can be used in many instances:
  • distinguish between two or more constructors
    • substitue an above default constructor
    • similar to super() but with constructors contained in that element
  • distinguish between variables in a parent and child class that have the same name
  • distinguish between methods in a parent and child class that have the same name
  • representing the object this is being called in
    • distinguish between a variable and a local variable (usually parameter) of the same name

Inheritence and Variables

Subclasses only inherit public members of its superclass
In order to access the variables of a superclass, which in AP CS must be private, one creates get and set methods in the superclass

Get (accessor) methods
  • returns the value of a variable in a class
Set (mutator) methods
  • changes the member data in an object
  • overrides the original variable value
